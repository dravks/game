# Game Dev

## Purpose

- This project is a 2D isekai MMORPG-style prototype, starting as a single-player experience with a city hub, NPC systems, dungeon progression, random class and starting stats, items, bosses, and character growth.
- Your role is to act as the main coding agent entry point and use this file as a lightweight router to the supporting project docs.

## Core Instructions

- Work incrementally. Do not try to build the whole game, architecture, or content set in one step.
- Preserve working systems. Avoid unnecessary rewrites, broad refactors, or changes that risk stable behavior without clear need.
- Before starting any implementation, read the related supporting docs when they exist and follow the guidance there.

## Document Map

- `WORKFLOW.md` - development flow, task order, and implementation sequence
- `PROGRESS.md` - current status, completed work, known issues, and next steps
- `DESIGN-DOCUMENT.md` - gameplay systems, scope, and feature design
- `GAME-ENGINE.md` - engine-specific technical guidance and patterns
- `ASSETS.md` - asset pipeline, organization, and usage rules
- `TESTING.md` - validation strategy, test expectations, and regression checks

## Working Rules

- Build in small validated steps.
- Do not over-engineer early.
- Keep code modular and expandable.
- Prefer playable results over theoretical planning.
- Record important progress and problems in `PROGRESS.md` once that file exists.

## Current Scope

- The project structure and workflow are being prepared first.
- Gameplay implementation is not the current step.
